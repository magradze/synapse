Header for the SD Card storage backend implementation.

This header declares the initialization and deinitialization functions for the SD Card backend. It is intended for internal use by the Storage Manager module only.

esp_err_t storage_backend_sd_init ( storage_api_t *  api_out)

Initializes the SD Card backend and populates the API function table.

This function configures the SPI bus, mounts the SD card using the FAT filesystem, and assigns the appropriate implementation functions to the provided API structure.

Parameters
[out]api_outA pointer to the storage_api_t structure that will be filled with function pointers for SD Card operations.
Returns
  • ESP_OK: If the SD Card backend was initialized successfully.
  • ESP_FAIL: If a critical error occurred during initialization.
