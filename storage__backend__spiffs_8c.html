Deinitializes the SPIFFS backend.

This function unmounts the SPIFFS partition, freeing up resources. It should be called by the Storage Manager during its deinit phase.

esp_err_t storage_backend_spiffs_init ( storage_api_t *  api_out)

Initializes the SPIFFS backend and populates the API function table.

This function mounts the SPIFFS partition and assigns the SPIFFS-specific implementation functions to the provided API structure.

Parameters
[out]api_outA pointer to the storage_api_t structure that will be filled with function pointers for SPIFFS operations.
Returns
  • ESP_OK: If the SPIFFS backend was initialized successfully.
  • ESP_FAIL: If a critical error occurred during initialization.
