Detailed Description

The public Service API structure for the System Manager.

This is the "contract" that the system_manager module implements and other modules (like command_router) consume.

esp_err_t(* system_manager_api_t::get_all_modules) (const module_t ***modules, uint8_t *count)

Retrieves a read-only list of all currently registered modules.

This function provides a snapshot of the module registry, allowing other modules to inspect the system's composition and module statuses. The memory for the returned list is managed by the module registry and should NOT be freed by the caller.

Parameters
[out]modulesA pointer that will be updated to point to an array of const module_t*.
[out]countA pointer to a variable where the number of modules in the array will be stored.
Returns
  • ESP_OK: If the list was retrieved successfully.
  • ESP_ERR_INVALID_ARG: If modules or count pointers are NULL.

esp_err_t(* system_manager_api_t::reboot_system) (void)

Initiates a graceful system reboot.

This function provides a standardized way to restart the device. It may perform cleanup operations before calling esp_restart().

Returns
This function does not return on success. It will return an error code if the reboot cannot be initiated.
