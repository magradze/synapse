Detailed Description

Implementation of the asynchronous Promise Manager.

Author
Synapse Framework Team
Version
1.0.0
Date
2025-08-29

This component manages the lifecycle of promises, providing a clean way to handle asynchronous request-response patterns. It uses a dedicated FreeRTOS task to execute callbacks, ensuring that they run in a safe, isolated context and do not block the callers.

◆ fmw_promise_destroy()

void fmw_promise_destroy ( promise_handle_t  handle)

Destroys a promise handle and releases its associated resources.

This function should be called by the consumer if it is no longer interested in the result of the promise. It signals to the promise manager that the resources can be cleaned up. The manager will also automatically clean up promises after they are executed.

Parameters
[in]handleThe promise handle to destroy. Can be NULL.

Rejects a promise with an error.

This action will queue the promise for execution, eventually triggering the 'catch' callback on the consumer's side. The promise manager takes ownership of the error_data pointer.

Parameters
[in]handleThe handle of the promise to reject.
[in]error_dataA pointer to the error data. Can be NULL.
[in]free_fnA function pointer to free the error_data. If NULL, standard free() is used.
Returns
  • ESP_OK: If the promise was successfully queued for rejection.
  • ESP_ERR_INVALID_ARG: If the handle is invalid or not found.
  • ESP_ERR_INVALID_STATE: If the promise is not in a PENDING state.

Resolves a promise with a successful result.

This action will queue the promise for execution, eventually triggering the 'then' callback on the consumer's side. The promise manager takes ownership of the result_data pointer and will free it according to free_fn.

Parameters
[in]handleThe handle of the promise to resolve.
[in]result_dataA pointer to the result data. Can be NULL.
[in]free_fnA function pointer to free the result_data. If NULL, standard free() is used.
Returns
  • ESP_OK: If the promise was successfully queued for resolution.
  • ESP_ERR_INVALID_ARG: If the handle is invalid or not found.
  • ESP_ERR_INVALID_STATE: If the promise is not in a PENDING state.
