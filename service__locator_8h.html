Detailed Description

Framework-ის სერვის ლოკატორის საჯარო API.

ეს კომპონენტი უზრუნველყოფს სერვისების რეგისტრაციისა და მოძიების ცენტრალიზებულ მექანიზმს. ის საშუალებას აძლევს მოდულებს, მიიღონ სხვა მოდულების API-ები პირდაპირი #include დამოკიდებულების გარეშე. ეს პატერნი ხელს უწყობს კომპონენტების იზოლაციას და სისტემის მოქნილობას.

ახალი სერვისის რეგისტრაცია უნიკალური სახელით და ტიპით.

ეს ფუნქცია ამატებს სერვისს ცენტრალურ რეესტრში. რეგისტრაცია არ განხორციელდება, თუ ამ სახელით სერვისი უკვე არსებობს. ოპერაცია ნაკად-უსაფრთხოა (thread-safe).

Parameters
[in]service_nameსერვისის უნიკალური სახელი (მაგ: მოდულის instance_name).
[in]service_typeსერვისის ტიპი fmw_service_type_t enum-იდან (მაგ: SERVICE_TYPE_DISPLAY_API).
[in]service_handleმაჩვენებელი სერვისის API სტრუქტურაზე.
Returns
esp_err_t ოპერაციის წარმატების კოდი.
Return values
ESP_OKსერვისი წარმატებით დარეგისტრირდა.
ESP_ERR_INVALID_ARGთუ service_name, service_handle არის NULL, ან service_type არავალიდურია.
ESP_ERR_NO_MEMთუ მეხსიერება ვერ გამოიყო ახალი სერვისისთვის.
ESP_ERR_INVALID_STATEთუ ამ სახელით სერვისი უკვე დარეგისტრირებულია.
ESP_ERR_TIMEOUTთუ mutex-ის დაკავება ვერ მოხერხდა.
