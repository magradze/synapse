◆ is_provisioning_active

bool(* ble_prov_api_t::is_provisioning_active) (void)

Checks if the BLE provisioning process is currently active.

A module can call this function to determine if the device is currently in a state where it is discoverable and waiting for a client to connect for provisioning.

Returns
  • true: Provisioning is active.
  • false: Provisioning is not active.

◆ start_provisioning

esp_err_t(* ble_prov_api_t::start_provisioning) (void)

Manually starts the provisioning process.

This can be used to re-trigger provisioning, for example, from a CLI command or a physical button press.

Returns
  • ESP_OK: If the provisioning process was started successfully.
  • ESP_ERR_INVALID_STATE: If provisioning is already active.
