Detailed Description

The public Service API structure for a display driver.

This is the "contract" that all display drivers implement. All functions take a void* context parameter, which allows them to operate on a specific display instance.

esp_err_t(* display_driver_api_t::draw_formatted_text) (void *context, int16_t x, int16_t y, uint32_t color, const char *format,...)

Draws a formatted text string to the internal buffer.

The driver should provide support for at least one basic font. This function behaves like printf.

Parameters

[in]xThe X coordinate for the start of the text.
[in]yThe Y coordinate for the start of the text.
[in]colorThe color of the text.
[in]formatA null-terminated, printf-style format string.
[in]...Variable arguments for the format string.
Returns
ESP_OK on success.

esp_err_t(* display_driver_api_t::draw_pixel) (void *context, int16_t x, int16_t y, uint32_t color)

Draws a single pixel to the internal screen buffer.

To reflect the change on the physical screen, update_screen() must be called.

Parameters
[in]contextThe service context.
[in]xThe horizontal coordinate of the pixel.
[in]yThe vertical coordinate of the pixel.
[in]colorThe color of the pixel. The format is driver-dependent (e.g., 0/1 for monochrome, RGB888 for color).
Returns
ESP_OK on success.
