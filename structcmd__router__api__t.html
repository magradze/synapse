Detailed Description

The public Service API structure for the Command Router.

This is the "contract" that the command_router module implements and other modules consume via the Service Locator.

◆ register_command

esp_err_t(* cmd_router_api_t::register_command) (const cmd_t *command)

Registers a new command with the router.

Parameters
[in]commandA pointer to a cmd_t structure describing the command. The pointed-to structure must remain valid for the lifetime of the registration.
Returns
  • ESP_OK: If the command was registered successfully.
  • ESP_ERR_INVALID_ARG: If the command pointer or its members are invalid.
  • ESP_ERR_NO_MEM: If the command list is full.
  • ESP_ERR_INVALID_STATE: If a command with the same name already exists.

esp_err_t(* cmd_router_api_t::unregister_command) (const char *command_name)

Unregisters a previously registered command.

This function should be called by a module in its deinit phase to clean up its registered commands.

Parameters
[in]command_nameThe name of the command to unregister.
Returns
  • ESP_OK: If the command was unregistered successfully.
  • ESP_ERR_NOT_FOUND: If no command with the given name was found.
