Detailed Description

Event Bus კომპონენტის API.

Version
1.0
Date
2025-06-24
Author
Giorgi Magradze

ეს კომპონენტი უზრუნველყოფს ასინქრონული შეტყობინებების (events) მართვის მექანიზმს. მოდულებს შეუძლიათ გამოიწერონ კონკრეტული ტიპის ივენთები და მიიღონ შეტყობინება, როდესაც რომელიმე სხვა მოდული გამოაქვეყნებს ამ ივენთს. ეს ხელს უწყობს მოდულების ერთმანეთისგან იზოლირებას.

esp_err_t event_bus_init ( void  )

Event Bus-ის ინიციალიზაცია და ფონური ტასკის გაშვება.

ეს ფუნქცია ქმნის ივენთების რიგს (queue) და აამუშავებს ფონურ ტასკს, რომელიც პასუხისმგებელია რიგში მოხვედრილი ივენთების დამუშავებასა და შესაბამისი გამომწერების ინფორმირებაზე. უნდა გამოიძახოს system_manager-მა ინიციალიზაციისას.

Returns
esp_err_t ოპერაციის წარმატების კოდი.
Return values
ESP_OKთუ ინიციალიზაცია წარმატებით დასრულდა.
ESP_FAILთუ ტასკის ან რიგის შექმნა ვერ მოხერხდა.

◆ event_bus_post()

esp_err_t event_bus_post ( system_event_id_t  event_id,
event_data_wrapper_t *  data_wrapper 
)

აქვეყნებს ივენთს Event Bus-ზე ასინქრონულად.

ეს ფუნქცია ამატებს ივენთს და მის თანდართულ მონაცემებს (თუ არსებობს) Event Bus-ის რიგში. ფონური ტასკი შემდგომში ამ ივენთს მიაწვდის ყველა იმ მოდულს, რომელსაც გამოწერილი აქვს ეს event_id.

Parameters
[in]event_idივენთის უნიკალური იდენტიფიკატორი.
[in]data_wrapperმაჩვენებელი "შეფუთულ" მონაცემებზე. თუ ივენთს მონაცემები არ სჭირდება, ეს პარამეტრი უნდა იყოს NULL. მნიშვნელოვანია: Event Bus არ არის პასუხისმგებელი ამ მეხსიერების განთავისუფლებაზე. გამომძახებელმა ან მიმღებმა უნდა იზრუნოს ამაზე.
Returns
esp_err_t ოპერაციის წარმატების კოდი.
Return values
ESP_OKთუ ივენთი წარმატებით დაემატა რიგში.
ESP_FAILთუ რიგი სავსეა ან სხვა შეცდომა მოხდა.
